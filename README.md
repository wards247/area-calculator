# 地块网格面积计算器

> 在地块规划图上描边 → 自动生成网格 → 滑动选格 → 实时计算面积。
> 支持比例分割（如 60/40、65/35），适用于农村地块面积估算场景。

## 快速使用

- **分享给别人**：直接发 `dist/area-calculator-standalone.html`（单文件，图片已内嵌，手机浏览器打开即用）
- **本地开发**：打开 `src/area-trace-dev.html`（需要 `assets/a24-map.png` 在相对路径）

## 功能

### 核心功能
1. **描边**：在地块图上点击描点，形成多边形边界，双击或点击起点闭合
2. **网格覆盖**：自动生成 20px 正方格子，用射线法判定格子中心点是否在多边形内
3. **滑动选格**：鼠标/触摸拖动选择格子，实时计算面积
4. **面积计算**：总面积 ÷ 总格数 = 单格面积；选中格数 × 单格面积 = 实际面积

### 比例填充面板（开发版）
- 描边完成后右侧自动弹出
- 三个预设比例：60/40、65/35、70/30
- 自定义比例输入
- 从左上角往右下角扫描，前 N 格标蓝色(A区)，后 M 格标橙色(B区)
- 点击格子可翻转 A/B 归属（不会清除比例）
- 实时显示实际占比、A/B 区格数和亩数

## 技术栈

纯前端，零依赖：HTML + CSS + Canvas + JavaScript。

## 技术要点

- **坐标映射**：图片坐标 ↔ Canvas 显示坐标，通过 `scale` 和 `offset` 双向转换
- **射线法**：从格子中心点向右发射射线，与多边形边相交次数为奇数则在内部
- **中心点采样**：用格子中心代表整格，将面积问题降维为点判定问题
- **单文件内嵌**：底图转 base64 嵌入 HTML，微信发送后直接打开

详见 `docs/grid-implementation-guide.md`。

## 已知设计决策

1. **网格大小固定 20px**：兼顾精度和手机触控体验，不使用四分小格细化
2. **中心点判定**：边界格子会有全或无误差，但大量格子统计下正负误差抵消
3. **总面积可编辑**：默认 1.12 亩，用户可手动修改
4. **比例模式不干扰纯选格**：描边后比例面板才出现，不用比例功能的人不受影响
